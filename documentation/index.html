OceanBeacon

Citizen Science Platform for Marine Pollution Reporting

OceanBeacon: Simply Explained

OceanBeacon is an app that lets regular people help fight ocean pollution by taking pictures of trash and contamination they find at beaches and coastal areas.

What it does

  • Anyone with a smartphone can take a photo of pollution they find
  • The app records where and when the photo was taken
  • Users categorize what kind of pollution it is (plastic, oil, chemicals, etc.)
  • All reports appear on a map showing pollution hotspots

Why it matters

  • Helps cleanup teams know where to focus their efforts
  • Provides scientists with real data about pollution trends
  • Shows government officials where environmental problems exist
  • Connects everyday people to ocean conservation efforts

How it works

  1. See pollution? Open the app and take a photo
  2. Select what type of trash/pollution you found
  3. Add a quick description if you want
  4. Submit your report in seconds
  5. View a map showing all reports from you and others

The app also includes simple guides to help you identify different types of pollution and understand their impacts on marine life.

By gathering data from thousands of people who visit beaches and coastal areas, we can create a powerful tool that turns individual observations into meaningful action for cleaner oceans.

1. Project Overview

OceanBeacon is a citizen science platform designed to enable individuals to document and report marine pollution. By leveraging crowdsourced data collection, the platform creates a comprehensive database of pollution incidents that can inform cleanup efforts, scientific research, and policy decisions. The platform utilizes mobile technology, cloud infrastructure, and data visualization to transform observations into actionable intelligence for environmental protection.

This two-week MVP development focuses on creating a functional core system that demonstrates the platform's potential while establishing a foundation for future expansion. The MVP prioritizes simplicity, usability, and data quality to ensure the platform delivers immediate value.

2. Mission and Vision

Mission

To empower communities worldwide to participate in marine conservation by documenting pollution and contributing to a global database that drives action and awareness.

Vision

A world where citizen scientists play a central role in marine conservation, where pollution data is transparent and accessible, and where this information directly influences cleanup efforts, research initiatives, and environmental policy.

OceanBeacon: Technical Stack & Implementation

Technology Overview

OceanBeacon leverages a modern, cloud-native architecture with Google ecosystem integration to provide a scalable, responsive platform for marine pollution reporting. Our stack is designed for rapid development while maintaining future extensibility.

Development Challenge: Maximize use of Google Cloud products while maintaining a flexible, modern architecture.

Frontend Technologies

Web Application

  • Framework: Vue.js with Nuxt.js
  • Benefits:
    • Server-side rendering for improved SEO and performance
    • Automatic code splitting for faster page loads
    • Vue's reactive data binding for responsive UI
    • Progressive Web App capabilities
  • UI Components: Vuetify (Material Design)
  • State Management: Pinia
  • Maps Integration: Google Maps JavaScript API

Mobile Application

  • Framework: Flutter
  • Benefits:
    • Cross-platform development (iOS & Android from single codebase)
    • Native-like performance
    • Rich widget library
    • Offline capabilities
  • State Management: Provider + Riverpod
  • Maps Integration: Google Maps Flutter plugin
  • Camera Access: Flutter Camera plugin

Backend Technologies

API Server

  • Framework: Express.js (Node.js)
  • ORM: Prisma
  • Database: PostgreSQL (with PostGIS extension for geospatial data)
  • Hosting: Google Cloud Run
  • API Documentation: Swagger UI

Authentication

  • Service: Firebase Authentication
  • Features:
    • Email/password authentication
    • Social login (Google, Facebook)
    • JWT token management
    • Security rules

Storage

  • Media Storage: Google Cloud Storage
  • Database: Google Cloud SQL (PostgreSQL)
  • Caching: Redis on Google Cloud Memorystore

AI/ML Integration

For pollution classification and validation, we'll implement a tiered approach:

Phase 1 (MVP - 2 Weeks)

  • Google Vision API for basic image classification and inappropriate content filtering
  • Manual verification by administrators
  • Simple rule-based validation

Phase 2 (Post-MVP)

  • Custom TensorFlow.js model for pollution-specific classification
  • Training data generated from validated user submissions
  • Deployment on Google Cloud AI Platform
  • Integration of Google AutoML for continuous model improvement

Technical Challenges & Solutions

Challenge 1: Offline Functionality

Solution: Implement service workers in the web app and local storage in Flutter to queue submissions when offline, syncing when connectivity is restored.

Challenge 2: AI Model Integration

Solution: Start with Google Vision API for MVP while collecting training data. Later develop custom model using TensorFlow and Google AutoML.

Challenge 3: Geospatial Queries

Solution: Use PostGIS extension with PostgreSQL on Google Cloud SQL for efficient spatial queries. Implement caching for common query patterns.

Challenge 4: Cross-Platform Consistency

Solution: Develop shared API contracts and validation logic. Implement feature flags to manage feature parity between web and mobile applications.

Challenge 5: Media Storage Optimization

Solution: Use Google Cloud Storage with image resizing and optimization. Implement efficient client-side compression before upload.
